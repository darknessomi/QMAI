# 配角配置

## 适用范围

配角、反派、导师、队友、家族成员、同门、同事、观众、工具角色升级。

生成或补全配角、反派、导师、盟友、上级、工具型配角时，必须遵循上级目录的 `CHARACTER_PROFILE_STANDARD.md`。配角文件要能说明这个角色为什么存在、如何推动剧情、如何制造情绪，以及何时退场或升级。

## 配角功能

- 资源型：给主角资源或入口。
- 压力型：制造阻碍和评价。
- 镜像型：映照主角可能走向。
- 情绪型：提供信任、背叛、陪伴、误解。
- 信息型：掌握线索或世界规则。
- 反派型：与主角目标冲突。

## 设计规则

1. 每个重要配角必须有自己的目标。
2. 配角不能只在主角需要时出现。
3. 反派层级要递升，小反派服务大反派或大规则。
4. 导师不能太强，也不能替主角解决关键问题。
5. 工具型配角只要反复出现，就必须补足独立目标、语言风格和状态追踪。
6. 反派必须记录攻击手段、资源层级、失败节点和是否有洗白空间。

## 章纲要求

- 写清配角本章推动了哪个核心事件。
- 如果配角只说话不改变局面，应删除或合并。
- 群像场景要给每个关键人一个行动功能。

## 输出要求

- 配角必须写清剧情功能和情绪功能。
- 反派必须写清利益动机、压迫方式、退场方式。
- 导师/上级必须写清权限资源、庇护或压制方式、不能替主角解决的问题。
- 情感型配角必须写清情绪触发器和主题连接。
