# 人物角色信息标准

## 使用范围

本标准用于 AI 大纲会话生成、补全、改写人物角色文件。角色文件不是普通人物小传，而是后续 AI 写章纲和正文时读取的“角色状态卡 + 语言风格卡 + 出场追踪卡”。

文件命名建议：

```text
角色/{角色名}.md
角色/男主/{角色名}.md
角色/女主/{角色名}.md
角色/男配/{角色名}.md
角色/女配/{角色名}.md
角色/反派/{角色名}.md
```

如果项目只有少量角色，可以先放在 `角色/{角色名}.md`；当角色数量增多时，再按男主、女主、男配、女配、反派、导师、盟友、上级、情感型配角等分类移动。

## 角色类型

| 类型 | 重点记录 |
|---|---|
| 男主/主角 | 显性身份、隐性身份、金手指、核心动机、当前能力、弱点、成长状态 |
| 女主/核心情感角色 | 身份背景、关系推进、情绪变化、隐藏背景、与主角互动方式 |
| 男配/女配 | 剧情功能、与主角关系、独立目标、资源或情绪作用 |
| 反派 | 利益动机、攻击手段、资源层级、压迫方式、失败节点、是否有洗白空间 |
| 导师/上级 | 权限资源、决策风格、庇护或压制方式、不能替主角解决的问题 |
| 盟友/队友 | 补位功能、独立行动、与主角的信任变化 |
| 情感型配角 | 情绪触发器、隐痛、主题连接、对主角造成的情绪影响 |
| 工具型配角 | 信息、资源、入口、冲突或反应层功能；重要工具人必须反工具化 |

## 生成规则

1. 人物角色文件必须服务写作，不允许只堆年龄、身高、外貌、家世等静态资料。
2. 每个重要角色必须写清“故事功能”和“情绪功能”，否则正文中容易变成无效出场。
3. 每个重要角色必须有“语言风格”，并标注面对不同对象时的说话差异。
4. 每次补全角色时必须更新“出场记录”和“状态追踪”，保证后续章纲和正文读取的是最新状态。
5. 角色行为必须能回到动机、信念、弱点或当前关系，不允许为了剧情需要突然变蠢、变善、变坏。
6. 反派必须有自己的利益或目标。短线小反派可以恶得直接，但仍要写清手段、收益和退场方式。
7. 配角必须有独立目标或明确功能。只说话、不改变局面、不给情绪作用的配角应删除或合并。
8. 情感型角色必须写清情绪触发器和关系阶段，不允许只写“喜欢主角”。
9. 缺少原文信息时写“原文未明确”或“待后续补充”，不能编造成已确认事实。

## 简版兼容

从旧人物小传导入时，常见字段为：身份、性格、能力、经历、关系、别名。导入后必须补齐：

- 故事功能
- 情绪功能
- 当前状态
- 核心动机
- 弱点/缺陷
- 语言风格
- 面对不同对象的说话差异
- 出场记录
- 状态追踪
- 写作使用规则
