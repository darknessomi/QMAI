# 伏笔与悬念

## 适用范围

伏笔、线索、隐藏身份、旧案、规则真相、反转、悬念节奏、回收计划。

输出伏笔或悬念追踪文件时必须遵循上级目录的 `SETTING_PROFILE_STANDARD.md`，重点写清伏笔状态、预计回收、回收方式、回收后果和过期处理。

## 生命周期

1. 埋设：自然出现，不解释。
2. 推进：重复出现但换角度。
3. 提示：回收前让线索汇聚。
4. 回收：产生“原来如此”的情绪。
5. 后果：回收后改变人物关系、规则或主线。
6. 追踪：每个伏笔必须有状态，不能只写灵感。

## 章纲要求

- 每章标注新增、推进、回收、延后处理的伏笔。
- 如果本章无新增伏笔，也要明确写“本章无新增伏笔”。
- 伏笔回收必须影响后续剧情，不能回收完就消失。

## 文件要求

- 伏笔默认输出到 `追踪/伏笔.md`。
- 状态使用：未埋 / 已埋 / 推进中 / 已回收 / 已过期。
- 过期伏笔必须标注补回收、延后、转长线或放弃并说明。
