# 设定信息标准

## 使用范围

本标准用于 AI 大纲会话生成、补全、改写小说设定文件。设定文件不是背景资料堆砌，而是后续章纲、人物和正文共同读取的规则约束库。

设定必须回答四件事：

1. 这个设定是什么。
2. 它如何影响人物行动。
3. 它如何制造冲突、爽点、悬念或情绪。
4. 它在写作时有哪些边界，哪些内容不能乱写。

项目规模小时可以合并文件；设定一旦影响多章正文，必须独立成文件。

## 设定类型

| 类型 | 默认文件 | 重点 |
|---|---|---|
| 题材定位 | `设定/题材定位.md` | 题材类型、核心梗、目标读者、卖点、风险 |
| 世界背景 | `设定/世界观/背景设定.md` | 世界差异、社会环境、时代背景、核心舞台 |
| 世界规则 | `设定/世界观/规则体系.md` | 规则来源、边界、代价、信息公开度、反制方式 |
| 力量体系 | `设定/世界观/力量体系.md` | 等级、能力、晋升、消耗、克制、代价 |
| 金手指 | `设定/世界观/金手指.md` | 绑定方式、任务机制、奖励、限制、使用边界 |
| 势力组织 | `设定/势力/{势力名}.md` | 目标、资源、内部矛盾、代表人物、主角关系 |
| 地图场景 | `设定/地图/{地图名}.md` | 环境规则、资源、危险、出入条件、阶段作用 |
| 道具物件 | `设定/道具/{道具名}.md` | 来源、用途、限制、归属、后续回收 |
| 伏笔悬念 | `追踪/伏笔.md` | 埋设、推进、提示、回收、后果 |
| 时间线 | `追踪/时间线.md` | 故事时间、事件顺序、并行线、疑点 |

## 生成规则

1. 设定必须写清“运行规则”和“写作约束”，不能只写背景介绍。
2. 新增设定必须服务主线、冲突、情绪、伏笔或人物变化。
3. 缺少来源的信息写“原文未明确”或“尚未设定”，不能编造成已确认事实。
4. 影响多章的设定必须有更新记录。
5. 金手指和力量体系必须写限制、代价和反制点，防止万能化。
6. 势力必须有目标、资源和代表人物，不能只是名字。
7. 地图变化必须改变行动规则、资源和压力，不能只是换地名。
8. 伏笔必须有生命周期：埋设、推进、提示、回收、后果。
9. 题材定位必须写清读者承诺，后续大纲、章纲、正文不得偏离核心卖点。
