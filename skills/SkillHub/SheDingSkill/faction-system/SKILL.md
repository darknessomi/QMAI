# 势力系统

## 适用范围

门派、家族、公司、官方机构、地下组织、反派阵营、宫廷、宗门。

输出势力、组织、阵营文件时必须遵循上级目录的 `SETTING_PROFILE_STANDARD.md`，重点写清势力目标、资源来源、内部矛盾、外部关系、代表人物和剧情功能。

## 设计清单

1. 势力目标：它当前最想得到什么。
2. 资源来源：金钱、武力、情报、人脉、地盘、制度、技术。
3. 内部矛盾：继承权、派系、利益差、理念差。
4. 外部对手：竞争、合作、互相利用或仇恨。
5. 主角关系：加入、利用、反抗、被追杀、继承因果。
6. 剧情功能：提供资源、压力、试炼、反转、伏笔或升级通道。
7. 更新记录：势力立场或资源变化必须记录。

## 章纲要求

- 势力登场必须推动一个核心事件。
- 势力要有代表人物和可见行动。
- 不要连续刷同类势力事件，至少保留节奏变化。

## 文件要求

- 独立势力默认输出到 `设定/势力/{势力名}.md`。
- 组织型势力也可输出到 `设定/组织/{组织名}.md`。
- 势力不能只有名字，必须有目标、资源和代表人物。
