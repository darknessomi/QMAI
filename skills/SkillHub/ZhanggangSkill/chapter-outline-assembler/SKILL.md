# 章纲汇总

## 核心原则

最终章纲要短到能直接拿来写，又完整到不容易写散。它必须显示本章发生什么变化、读者感受到什么、读者记住什么画面，以及为什么继续看下一章。

汇总时必须对齐上级目录的 `CHAPTER_OUTLINE_STANDARD.md`。如果输入组件只包含旧版简版细纲字段，先补齐缺失项，再输出章纲。

## 必要输入

- 上章承接和当前状态
- 章节定位
- 浓缩剧情和关键词条件
- 核心事件因果链
- 四段式章纲
- 情绪曲线
- 爽点与看点
- 关键信息与扩写方式
- 画面细节
- 伏笔和钩子方案
- 角色状态变化
- 设定/世界观/道具更新
- 下一章交接

缺少某一部分时，先补齐或合成一版“假设稿”，并明确标注假设来源。

## 执行步骤

1. **合并时消除矛盾**
   - 章节属性、情绪曲线、核心爆点、结尾钩子必须指向同一个阅读效果。

2. **保持可写作**
   - 使用具体场景指令，不写抽象建议。
   - 保留关键词、必要条件和必须插入线索，让写作者有材料可落笔。

3. **检查章节价值**
   - 本章至少有一个可见变化。
   - 本章至少有一个记忆点。
   - 本章必须有一个继续阅读的理由。

4. **处理连续章节交接**
   - 如果输出多章，每章末尾都写明“交给下一章的问题”。
   - 下一章开头必须承接上一章钩子，不能另起一套无关冲突。

## 输出格式

```
# 章纲-第001章：{章节标题}

## 1. 基础信息

## 2. 上章承接

## 3. 本章定位

## 4. 浓缩剧情

## 5. 核心事件链

## 6. 关键词与必要条件

## 7. 四段式章纲

## 8. 情绪曲线

## 9. 爽点与看点

## 10. 关键信息与扩写方式

## 11. 画面细节

## 12. 伏笔与钩子

## 13. 出场角色与状态变化

## 14. 设定/世界观/道具更新

## 15. 写作约束

## 16. 下一章交接

## 17. 写作检查清单

- [ ] 三行内入戏
- [ ] 本章有变化
- [ ] 核心事件链有因果
- [ ] 一个核心记忆点
- [ ] 至少一个动作微细节
- [ ] 至少一个爽点/看点/情绪触动点
- [ ] 回收一个旧信息或旧问题
- [ ] 埋下一个新信息或新问题
- [ ] 结尾有明确追读钩子
- [ ] 已写明下一章交接
```

## 边界

- 不写完整正文，除非用户明确要求。
- 不扩展成全书大纲。
- 不隐藏缺失组件；缺什么就标注假设。
- 不输出只有摘要、没有行动指令的章纲。
