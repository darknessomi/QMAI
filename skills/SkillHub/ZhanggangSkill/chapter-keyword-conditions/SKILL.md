# 浓缩剧情和关键词条件

## 核心原则

可写作的章纲必须有“浓缩剧情 + 关键词 + 必要条件”。章节定位决定这一章的功能，关键词条件决定这一章里面必须出现哪些可写材料。

## 执行步骤

1. **写浓缩剧情**
   - 把本章压缩成一句清楚的剧情句。
   - 句式可参考：“角色因为某个条件进入某个场景，在某个意外或冲突中产生某个变化。”

2. **提取 3-6 个关键词**
   - 可包含：场景、事件、情绪、道具、关系、特殊画面。
   - 避免“精彩”“推进”“铺垫”这类空泛词。

3. **补齐必要条件**
   - 入场条件：角色为什么必须进入这个场景。
   - 触发条件：什么意外、压力、邀请、陷阱或发现启动事件。
   - 互动条件：角色如何相遇、碰撞、误会、合作或对抗。
   - 结果条件：本章结束时发生什么变化。

4. **判断套路或反套路**
   - 如果使用常见套路，要明确读者期待的爽点或情绪回报。
   - 如果反套路，要明确用什么替代原本的期待回报。

5. **分配必须插入的信息**
   - 可加入线索、道具、台词、目击者、伤口、雨、信件、信物、期限、背景事实。
   - 只保留会影响剧情、情绪、钩子或后续回收的信息。

## 输出格式

```
## 浓缩剧情和关键词

### 浓缩剧情

### 关键词

### 必要条件

| 类型 | 内容 | 作用 |
|---|---|---|

### 套路或反套路处理

### 必须插入的伏笔/道具/信息
```

## 常见问题

- 有章节功能，但没有具体可写的场景材料。
- 关键词太多，导致本章失焦。
- 道具或线索出现后没有后续用途。
- 意外事件只服务作者安排，没有角色动因或现场逻辑。
