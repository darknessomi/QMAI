# 伏笔和钩子

## 核心原则

一章最好回收一个旧坑、埋下一个新坑、抛出一个结尾钩子。伏笔可以藏在中段，钩子必须在结尾让读者看见。

## 三个动作

1. **回收旧伏笔**
   - 解决或部分解释前文的线索、承诺、冲突或疑问。
   - 回收要让读者觉得“原来如此”，而不是更困惑。

2. **预埋新伏笔**
   - 把后续要用的线索藏在物件、动作、对话、环境或异常细节里。
   - 现在不必完全显眼，但以后必须能回收。
   - 优先使用章节关键词或必须插入信息里的线索。

3. **结尾强钩子**
   - 结尾留下未完全解决的事：发现、威胁、矛盾、邀请、新身份、失踪、突发证据、不可能事实。
   - 读者应该清楚知道自己下一章想看什么答案。
   - 钩子必须从本章结果里长出来，不能像额外贴上的段落。

## 钩子规则

- 结尾必须有缺口、未知或未完成结果。
- 视角角色已经知道答案时，不要制造假悬念。
- 钩子不是“发生了点事”，而是一个具体未答问题。
- 伏笔和钩子不同：伏笔可以隐，结尾钩子必须拉人。

## 输出格式

```
## 伏笔和钩子

### 回收旧伏笔

### 预埋新伏笔

### 结尾强钩子

### 读者会问的问题
```

## 常见问题

- 忘记前面埋过的线索。
- 结尾太圆满，没有追读动力。
- 新线索没有未来用途。
- 需要追读时，却用睡觉、休息、离场或场景关闭收尾。
