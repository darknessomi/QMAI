# 四段式章纲

## 核心原则

用四段锁住单章节奏：开篇入戏、中段冲突、核心爆点、强钩收尾。每一段都有任务，不让章节滑成松散对话或说明书。

## 输入

- 章节定位
- 浓缩剧情
- 关键词
- 必要条件
- 必须插入的线索、道具或信息

## 四段结构

| 段落 | 默认篇幅 | 任务 |
|---|---:|---|
| 开篇入戏 | 500-800 字 | 承接上一章结尾状态，快速落到场景、人物处境和遗留矛盾 |
| 中段冲突 | 500-800 字 | 事件升级，矛盾显现，对手施压，现场发生变化 |
| 核心爆点 | 500-800 字 | 本章最高光时刻：打脸、反转、揭秘、崩溃、觉醒、胜利或线索突破 |
| 强钩收尾 | 500-800 字 | 情绪收束，主角状态更新，确定下一步行动，留下下一章钩子 |

篇幅只是默认参考；用户指定字数时，按比例压缩或扩展。

## 执行步骤

1. **三行内入戏**
   - 快速进入场景，不做装饰性热身。
   - 使用 `chapter-keyword-conditions` 的入场条件。

2. **中段必须改变局面**
   - 压力增加、信息改变、关系变化或有人采取行动。
   - 使用触发条件和互动条件制造变化。

3. **给本章一个记忆点**
   - 读者应该记住一个强画面、揭示、胜利、损失或反转。
   - 从最强关键词或必须插入的信息里选择记忆点。

4. **结尾制造继续阅读的理由**
   - 结尾留下具体未解决结果，而不是模糊的“欲知后事如何”。
   - 钩子要连到结果条件或必须插入的线索。

## 输出格式

```
## 四段式章纲

### 1. 开篇入戏

### 2. 中段冲突

### 3. 核心爆点

### 4. 强钩收尾
```

## 常见问题

- 开头铺设太久，迟迟不入戏。
- 中段只重复争吵，没有新压力或新信息。
- 核心爆点没有视觉记忆点。
- 结尾把问题全部解决，读者没有继续看的理由。
