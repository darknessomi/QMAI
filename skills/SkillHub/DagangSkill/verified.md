# 验证通过的方法论单元

## 通过单元

### v01 - 网文大纲主控流程

type: workflow

- V1 跨域：通过。音频多次强调“不要复杂模板”“按公式一步步写”“最后进入黄金三章”。
- V2 预测力：通过。遇到任何“我想写小说但没有大纲”的场景，都能推导出下一步应该先收集剧情骨架，而不是直接堆设定。
- V3 独特性：通过。它的反直觉点是“世界观最后写”，而不是常见的“先搭庞大设定”。

### v02 - 500 字剧情骨架

type: technique

- V1 跨域：通过。剧情作为第一步，同时为卖点、目标、人设、世界观提供输入。
- V2 预测力：通过。给一个模糊创意时，可以推导出必须补齐主角、环境、机会、方向、冲突。
- V3 独特性：通过。不是写详细章纲，而是先写低成本可扩展骨架。

### v03 - 核心卖点提炼

type: framework

- V1 跨域：通过。音频用“别人都写修仙，凭啥看你的”和“泡面口味”两次说明差异化。
- V2 预测力：通过。可以判断一个创意是否只是同题材复刻，逼出可展示的钩子。
- V3 独特性：通过。强调“值得拿出来卖”的具体钩子，而不是泛泛说要创新。

### v04 - 阶段目标阶梯

type: framework

- V1 跨域：通过。目标承接剧情，并反过来让主线清晰。
- V2 预测力：通过。可用于任何长篇连载，按篇幅或篇章生成升级目标。
- V3 独特性：通过。不是列事件，而是用阶段性征服目标驱动长线。

### v05 - 人设服务剧情

type: principle

- V1 跨域：通过。主角人设和配角角色都围绕剧情需要。
- V2 预测力：通过。能从“灵虫代战”推导出“不热血、聪明、很狗”的主角。
- V3 独特性：通过。不是先捏人设，而是从玩法和剧情反推角色。

### v06 - 世界观后置

type: principle

- V1 跨域：通过。音频先警告新人爱堆世界观，再要求根据前面内容设定。
- V2 预测力：通过。能判断哪些设定该写，哪些设定只是拖延开写。
- V3 独特性：通过。反常见新人习惯：不是先设定等级体系，而是最后补必要规则。

### v07 - 黄金三章检查

type: quality-gate

- V1 跨域：通过。音频明确把前面步骤导向黄金三章。
- V2 预测力：通过。能判断大纲是否已经足够支持开篇，而不是继续补设定。
- V3 独特性：通过。把大纲的完成标准绑定到开篇可写性，而非文档完整性。
