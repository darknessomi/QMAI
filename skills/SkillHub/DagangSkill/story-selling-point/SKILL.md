# Webnovel Selling Point

## Core Principle

The selling point answers: “同题材这么多，读者凭什么看这一本？” It must grow from the plot, not be pasted on as a random gimmick.

## Execution

1. **Name the category baseline**
   - State what readers normally expect from this genre.

2. **Find the twist**
   - Change the power source, method, relationship, constraint, or perspective.
   - Example pattern: not “打怪升级”, but “养灵虫代替主角作战”.

3. **Turn it into repeatable scenes**
   - A selling point is usable only if it can generate many scenes.

4. **Reject weak hooks**
   - If the hook can be said about any novel, revise.

## Output Format

```markdown
## 核心卖点

### 同类作品默认写法

### 本书差异化钩子

### 可反复出现的爽点场景

### 一句话卖点
```

## Boundary

- Do not force novelty that breaks the user’s chosen genre promise.
- Do not add five hooks. Pick one main hook and 1-2 supporting hooks.
