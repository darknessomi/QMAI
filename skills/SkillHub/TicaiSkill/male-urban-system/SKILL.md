# 男频都市脑洞系统流

## 适用范围

都市脑洞、系统流、签到、加点、模拟、错位系统、职业流、金手指驱动故事。

## AI 大纲路由约束

- 使用本 Skill 前必须先读取上级目录的 `GENRE_ROUTE_STANDARD.md`。
- 必须先输出题材卡，再进入总纲、章纲、人物或设定生成。
- 本 Skill 只服务 AI 大纲体系，不处理正文协作区的正文生成。

## 核心读者期待

- 核心梗一句话能说清。
- 系统不是外挂说明书，而是制造情绪缺口和爽点循环的发动机。
- 每章至少有期待点、爽点或期待到爽点之间的推进。
- 主角使用系统解决现实压力，并引发更大压力。

## 大纲生成规则

1. 先定义系统核心玩法：触发条件、奖励类型、限制、惩罚、升级方式。
2. 明确核心循环：压力出现 → 系统触发 → 主角选择 → 爽点兑现 → 新问题扩大。
3. 系统功能不要过多，首卷只保留 1 个主能力和 1 个辅助能力。
4. 微创新不超过 3 个，优先在人设、关系、情节里创新，不乱改底层题材期待。
5. 每卷扩展系统边界，但不能破坏最初卖点。

## 章纲生成重点

- 本章要写清系统触发点和现实压力。
- 奖励必须改变主角处境，不能只是堆数据。
- 反派或旁观者要产生认知差：主角觉得正常，别人觉得不可思议。
- 章尾钩子优先：新任务、新限制、奖励副作用、系统来源线索。
