# 校验与验证分支说明

## 分支用途

本分支用于开发正文初稿生成后的"校验与验证"环节。该环节属于 AI 会话工作流中"写后自检"链条的一个独立子 skill，专门处理硬偏差（角色认知、状态、承接、伏笔冲突），风格 / 节奏等软偏差不在本分支范围内。

## 独立原因

按项目 `AGENTS.md` 第四章要求，新增功能必须放到独立功能分支中开发，避免污染 main 分支。本分支只做校验 skill + 校验面板 UI + 工作流接入三件事，不在本分支内混入其它功能。

## 使用要求

1. 本分支必须保持可打包、可运行源码状态。
2. 旧功能不回退是硬指标：续写、深度生成、大纲生成、AI 会话所有意图、审查中心、记忆中心等必须保持现有行为。
3. 未确认草稿绝不写入正式记忆库——校验阶段只读记忆库做比对，不得改写记忆库（项目 `.agents/AGENTS.md` 一致性原则）。
4. 校验阶段所有面向用户的提示语、UI 文案、按钮文字必须中文。

## 分支关键决策汇总

| 维度 | 决策 |
|---|---|
| 触发时机 | 正文初稿生成完后自动触发 |
| 修复策略 | 先报告偏差点，再自动修 |
| 重校策略 | 只校验被改动相关的项，最多 2 轮防死循环 |
| 校验真源 | 角色认知 + 状态 + 关系 + 伏笔（不含 soul.md 风格） |
| Skill 定位 | 写后自检的子 skill，专注硬偏差 |
| UI 形态 | 主区域新增 Tab：对话 \| 校验 |
| Tab 切换 | 校验进行中锁切回；完成后解锁 |
| 校验完成去向 | 停留校验 Tab，给"接受/拒绝/重校"按钮，保存仍走对话 Tab 现有按钮 |
| 取数策略 | 默认走记忆中心派生数据；内部矛盾时才读原始上一章正文做仲裁 |
